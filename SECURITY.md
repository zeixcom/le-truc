# Security Policy

## Supported Versions

| Version | Security updates |
| ------- | ---------------- |
| 2.x     | ✅ Yes           |
| 1.x     | ✅ Yes           |
| 0.x     | ❌ No             |

We support the two most recent major versions. Older major versions do not receive security updates.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report vulnerabilities by email to **support@zeix.com**. Include a description of the issue, steps to reproduce, and the potential impact.

You can expect an acknowledgement within **72 hours**. We are committed to releasing a fix within **10 days** of confirmed disclosure.

We follow responsible disclosure: once a fix is released, we will publish a security advisory. We will not take legal action against researchers who report vulnerabilities in good faith.
