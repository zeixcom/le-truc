Function: bind()#

bind<T, P, E, K>(type, key, reactive?): Effect<P, E>

Defined in: src/effects/event.ts:124

Two-way binding between a host signal property and a DOM element property.

Combines two coordination pieces that are easy to get wrong separately:

  1. Outward (signal → DOM): a raw createEffect() that writes directly to the DOM property, bypassing updateElement's DOM-read early-exit. updateElement bails via Object.is when the signal value equals the current DOM property — which happens immediately after a native browser interaction sets the property before the effect runs, breaking repeated toggles after the first.
  2. Inward (DOM → signal): an event listener that writes back to the signal inside batch(), deferring the flush until after the handler returns. This lets delegated listeners higher up the tree fire first (and update their own signals), so that when the flush finally runs the outward effect it sees the correct final value rather than a stale intermediate one.
