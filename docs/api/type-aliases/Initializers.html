Defined in: src/component.ts:80

The props argument of defineComponent — a map from property names to their initializers.

Each value may be:

  • A static value or Signal — used directly as the initial signal value.
  • A Parser (two-argument function branded with asParser()) — called with (ui, attributeValue) at connect time and again on every attribute change.
  • A Reader (one-argument function) — called with ui at connect time; if it returns a function or TaskCallback, a computed/task signal is created; otherwise a mutable state signal is created.
  • A MethodProducer (branded with asMethod()) — called for side effect of creating the method only; its return value is ignored.
